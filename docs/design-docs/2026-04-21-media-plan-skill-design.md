# 通用型传媒策划方案 Skill 设计

## 状态

- 状态：implemented
- 日期：2026-04-21
- 适用范围：中国市场、提案级、Markdown 正文交付

## 背景

目标是为一个面向传媒策划场景的 skill 体系定义首版设计，使用户在提供半结构化 brief 后，可以自动完成关键缺口补问、近期调研、洞察提炼、创意发散、执行规划、方案写作与回退评审，最终产出一份可直接继续打磨的完整传媒策划方案正文。

这个 skill 首版不追求覆盖所有策划细分场景，而是优先做好通用型方案生产流程，后续再向品牌传播、活动策划、新品上市、政企宣传等垂类扩展。

## 目标

- 接收半结构化 brief，并校验关键信息是否齐全。
- 强制执行实时调研，优先使用近期公开信息、热点、案例与数据源。
- 基于调研结果生成多方向创意，而不是直接给单一路径答案。
- 形成提案级完整方案正文，至少覆盖背景、洞察、策略、创意、执行、节奏、KPI、风险等章节。
- 在数据不足、创意老套、上下游脱节时触发回退 loop，而不是带着缺口硬写成案。

## 非目标

- 首版不直接生成 PPT、docx 或视觉排版文件。
- 首版不细化到周级执行排班、岗位分工或供应商管理。
- 首版不覆盖海外市场信源与国际传播语境。
- 首版不尝试做垂类行业专用模板，只保留通用扩展位。

## 输入约束

用户默认采用半结构化 brief 输入，理想字段包括：

- 项目背景
- 核心目标
- 目标受众
- 时间周期
- 预算或可用资源
- 地域范围
- 渠道限制或媒介偏好

在以下字段未补齐前，系统不得进入正式调研：

- 目标
- 受众
- 时间
- 预算或资源边界
- 地域
- 渠道限制

在进入执行规划与最终成案前，还必须补齐以下执行关键字段：

- 交付物需求
- 额外约束
- 成功指标

## 输出约束

首版默认输出为 Markdown 正文，颗粒度为提案级，要求包括：

- 项目背景与问题定义
- 市场与行业洞察
- 目标与受众定义
- 传播策略与核心主张
- 创意方向与亮点机制
- 执行路径与渠道动作
- 阶段节奏与排期建议
- KPI 与评估方式
- 风险与预案

## 推荐架构

采用 `1 个总控 skill + 4 个阶段子 skill + 4 份共享 rubric/template` 的混合架构。

### 1. 总控 skill

`media-plan-orchestrator`

职责：

- 接收用户 brief
- 判断输入完整度
- 触发关键缺口补问
- 驱动阶段流转与回退
- 控制最终成案节奏

### 2. 阶段子 skill

`industry-insight-research`

- 采集近期热点、行业趋势、竞品动作、案例与公开数据
- 区分已验证事实、可引用来源和分析推断
- 输出事实包与机会点

`strategy-ideation`

- 从调研中提炼洞察与传播命题
- 发散多个创意方向并进行筛选
- 用反老套规则淘汰平庸创意

`execution-planning`

- 将策略翻译为阶段节奏、渠道动作、资源建议、KPI 与风险控制
- 保证执行建议与策略主张一致

`proposal-writing-review`

- 组织完整方案正文
- 检查章节缺项、证据不足和逻辑脱节
- 把缺口回退到上游阶段修补

### 3. 共享文件

- `brief-intake-template.md`：半结构化 brief 字段定义与提问口径
- `research-rubric.md`：调研充分性判断标准
- `idea-scorecard.md`：创意新鲜度、差异化与可执行性评分标准
- `proposal-template.md`：方案正文固定结构与写作顺序

这些 shared 文件是仓库维护时的 canonical contract；如果宿主运行环境只复制单个 skill 到临时目录，则允许把运行所需的最小 contract 镜像进对应 `SKILL.md`，但两边必须同步维护，避免漂移。

## 工作流

### 主流程

`收 brief -> intake 前预搜索 -> 校验完整度 -> 补问关键缺口 -> 实时调研 -> 洞察提炼 -> 创意发散与筛选 -> 策略成型 -> 执行设计 -> 方案写作 -> 成案评审`

- 如果用户已经给出可识别的品牌、品类、平台、议题或地域信号，总控应先做一轮轻量预搜索，用于缩小补问范围并保留后续 research 线索。
- 预搜索产出的内容只能作为“待确认上下文”，不能直接顶替必填 brief 字段，也不能绕过 intake gate。

### 回退规则

- 数据不足、证据弱、缺少近期来源：回到 `实时调研`
- 创意方向太常规、缺少传播记忆点：回到 `创意发散与筛选`
- 策略主张与执行动作不一致：回到 `策略成型` 或 `执行设计`
- 方案章节空泛、缺少论据或缺章节：回到对应上游阶段补料

### 终止条件

满足以下条件后才允许输出“完成方案”：

- 关键 brief 字段已齐全
- 执行关键字段已齐全
- 调研包含近期公开信息与来源
- 创意至少给出 3 个方向并完成筛选
- 已存在显式的 Selected Direction Handoff
- 已存在通过验证的 Validated Execution-Plan Handoff
- 最终方案结构完整
- 评审未发现明显证据缺口或逻辑断裂

## 质量门槛

### 调研门槛

- 必须包含近期热点、行业案例或公开数据中的至少两类证据。
- 结论必须区分事实、解释和推断。
- 不允许把模型常识包装成“已完成调研”。

### 创意门槛

- 至少发散 3 个方向，再筛出推荐路径。
- 推荐方向需同时满足新鲜度、相关性、可执行性三个维度。
- 若创意只是常见话术重组，视为不通过，必须回炉。

### 成案门槛

- 每个核心章节必须有明确结论，不接受空泛标题。
- 执行部分必须能支撑策略，不允许只有口号没有动作。
- KPI、风险与预案必须出现，不能缺省。

## 工具绑定

首版绑定当前 Agent 环境，直接依赖以下能力：

- 当前 Agent 的交互式澄清能力：补问 brief 缺口与关键决策
- `websearch`：搜索近期热点、公开数据、行业信息
- `webfetch`：抓取指定来源内容
- `read` / `glob`：读取本地模板、rubric 与仓库文档

如果未来要迁移到别的平台，应再抽象一层“能力接口说明”，而不是在首版阶段先做平台无关化。

## 验收场景

### 场景 1：brief 缺口明显

输入只提供项目背景和大致目标时，总控 skill 应先结合可识别信号做轻量预搜索，再补问关键缺口，而不是直接输出方案。

### 场景 2：调研证据不足

当只拿到泛化描述、没有近期公开来源时，系统应继续搜索并补足来源，而不是宣布调研完成。

### 场景 3：创意过于保守

当产出的创意只是常规传播玩法堆叠时，系统应基于 scorecard 判为不通过，并回到 ideation 阶段继续发散。

### 场景 4：执行与策略脱节

如果策略主张强调年轻化互动，但执行动作仍是传统单向投放，评审阶段应打回重做。

## 风险与后续

- 最大风险不是文案质量，而是 loop 失效，导致系统带着缺口继续输出。
- 如果调研源质量不稳定，方案会快速退化成泛化模板，因此 research rubric 必须尽早成型。
- 如果 ideation 没有清晰评分卡，所谓“创意升级”很容易变成主观判断。
- 首版完成后，应优先做压力测试，再决定是否扩展到 Word/PPT 输出。
