# Proposal Phase

## Goal

Assemble the final proposal and review it against the shared template and upstream evidence chain before delivery.

## Required Read

- `../shared/highlight-mechanism-canvas.md`
- `../shared/proposal-template.md`

## Rules

- Start only when all proposal entry conditions pass.
- Pull inputs from the approved intake summary, research pack, scored idea evaluation, and validated execution-plan handoff.
- Pull the selected direction's `## Highlight Mechanism Canvas` forward explicitly; do not re-invent the theme or hero event while writing.
- Include the `## 9. Phased Event Menu` section per `../shared/proposal-template.md`. Each table row should trace to `Pillar roster by phase` in `## Validated Execution-Plan Handoff`; if a row is dropped or renamed, state the reason in `Risks And Open Questions` or in a one-line note under the table.
- Keep every section decision-useful: conclusion, reason, implication.
- Keep the recommendation visibly traceable from brief to research to idea choice to execution plan.
- If evidence, idea quality, or execution fit is unresolved, stop and route back upstream rather than patching with filler.
