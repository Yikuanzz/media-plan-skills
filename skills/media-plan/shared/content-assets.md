# Content Assets

Long-, mid-, and short-form **named** content lines that the campaign can **repeat** and **measure** — not a duplicate of the Event Menu.

## When required

- **Ideation** (end): After `## Highlight Mechanism Canvas` for `keep` directions, output `## Content Assets` for the primary direction (co-directions optional).
- **Proposal**: Section `Content Assets` (see [proposal-template.md](proposal-template.md)) must echo the same three tiers; Event Menu rows should **link** to an asset name where applicable.

## Output shape

```md
## Content Assets

### Long-form (e.g. documentary, deep feature, long video series, report)
- Title / working name:
- Purpose and key proof:
- Primary platform or distribution:
- When (phase window):

### Mid-form (e.g. fixed column, IP season, live show series, podcast)
- Title / working name:
- Structure (episode count or cadence):
- Primary platform:
- When (phase window):

### Short-form (e.g. UGC challenge, KOS tasks, memes, snackable cutdowns)
- Program / task name:
- User or creator action:
- Primary platform:
- When (burst vs sustained):
```

## Rules

- **At least one** named line per tier for the primary `keep` direction. If a tier is truly not used, mark `N/A` with one-sentence justification.
- Names must be stable enough to appear in the **Event Menu** “link to main idea / hero” column.
- Tiers must not all describe the same single hero event — differentiate length and **job-to-be-done** (e.g. depth vs always-on feed).
